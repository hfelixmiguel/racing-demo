## Technical Decisions Log

### 2026-03-07
1. **Architecture Pattern**: ECS-inspired modular structure chosen for:
   - Easy component addition/removal
   - Clear separation of concerns
   - Better code organization and maintainability

2. **Physics Approach**: Using delta time for all physics calculations to ensure:
   - Frame-rate independence
   - Consistent behavior across different hardware
   - Smooth gameplay experience

3. **Track Configuration Format**: JSON files chosen because:
   - Easy to edit without recompiling code
   - Human-readable format
   - Flexible structure for future track types

4. **Asset Strategy**: Starting with simple shapes (rectangles, circles) to:
   - Avoid external asset dependencies
   - Keep project self-contained
   - Enable quick prototyping and iteration

5. **Flask/Vercel Integration**: WSGI entry point required because:
   - Vercel needs explicit entry point for Flask apps
   - wsgi.py provides the WSGI application callable
   - .vercel/config.json defines build command and output directory
